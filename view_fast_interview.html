Fast интервью

Пять безопасных входных вопросов, которые не заставляют клиента уходить в подробный аудит, но позволяют автоматически предзаполнить часть основного интервью. Внешний Apps Script не меняем: на сохранение уходит тот же payload, а под капотом мы считаем score, переводим его в да / частично / нет и записываем только поля основного интервью.
Все вопросы ниже поддерживают мультивыбор и свободный комментарий. Комментарии сохраняются в payload записи, но не ломают существующую структуру листа и скрипта.

Как можно описать вашу ИТ-среду сегодня?

Можно выбрать несколько вариантов. Это влияет на сигналы по охвату, discovery и единому источнику.
Q1

Если нужно быстро понять, какие устройства и ПО реально есть в инфраструктуре, откуда обычно берут информацию?

Вопрос про реальные источники данных. По ответам считаются сигналы T1 / T2 / T3 / T5 / T8 / P4.
Q2

Какие системы управления ИТ используются сейчас?

Выясняем зрелость контура и то, куда уже потенциально могут уходить данные об инвентаризации.
Q3

Используются ли инструменты автоматического обнаружения инфраструктуры?

Отсюда считаются сигналы по T2 / T3 / T4 / T5. Если discovery есть хоть в каком-то виде, раскроется уточнение по инструментам.
Q4
Уточнение (если discovery есть)
Какие примерно это инструменты? Можно выбрать несколько вариантов.

Когда нужно быстро понять состав инфраструктуры, сколько времени обычно занимает собрать актуальную картину?

Этот вопрос стабилизирует оценку T1 / T2 / T5 и помогает не завышать зрелость только потому, что у клиента где-то есть отдельная система.

Предзаполнение
score → ответ
Порог: ≥3 = Да, от -2 до 2 = Частично, ≤-3 = Нет.
Fast Interview теперь не пишет боли / причины / потребности напрямую. Он только рассчитывает ответы основного интервью. Это убирает конфликты и сохраняет текущую цепочку: интервью → индексы → тематики → боли / потребности → выводы.
